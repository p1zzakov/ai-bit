# AI-BIT Enterprise 2.0.0-alpha.2

## Automatic Capability Discovery

Release adds evidence-based automatic discovery of corporate capabilities before Reference Model Audit.

The engine analyses the latest available crawl, Business Architecture, Process Mining and Operational Intelligence artifacts. It searches for evidence of smart processes, business-process templates, approval routes, internal request forms and signs of actual process use.

Statuses:

- `implemented` — multiple independent evidence signals were found;
- `partial` — a capability or related object was found, but evidence of a complete working process is insufficient;
- `unknown` — reliable evidence was not found; this status is never treated as `missing`;
- `missing` — used only for explicitly confirmed gaps or reliable negative evidence.

Confirmed company gaps remain authoritative and cannot be overridden by heuristic discovery:

- electronic document exchange;
- contract approval;
- user provisioning request for AD and 1C;
- internal memos.

## API

```bash
curl -sS -X POST http://127.0.0.1:8090/capability-discovery/collect | jq
curl -sS http://127.0.0.1:8090/capability-discovery/latest | jq
curl -sS -X POST http://127.0.0.1:8090/reference-audit/collect | jq
```

Reference Audit capability rows now include `source`, `confidence` and discovered evidence. The Executive Brief at `/#management` receives updated reference coverage automatically through Executive Intelligence.
